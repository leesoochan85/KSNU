Q: c4474 'function': 형식 문자열에 전달된 인수가 너무 많습니다.
Solve: scanf 형식지정문자 쓸 때 ""안 공백 없어야 함

Q: scanf 형식 지정문자 여러개를 한 번에 쓸 때
S: ""를 따로 사용하지 않고 %d를 띄어서(또는,같은 기호 사용) 갯수만큼 작성 (ex: "%d %d",&a,&b)

Q: double로 형식을 지정해주었을 때
S: float의 형식지정을 scanf()에 넣으면 값이 달라진다

Q: int 형으로 형식을 받고 프린트문에서 형식변화 하는법
S: (double)a이런식으로 변경 가능

Q:문자열 배열에서 주소의 값마다 같은지 확인해주는 자료형 :strcmp
S:문자열 끝에 0이 나오고 동시에 0이 나오면 같다고 판별 같으면 0 다르면 1 or -1
S:배열에서는 == 사용 불가, scanf에서 이미 주소를 나타내므로 &를 사용할 필요없다.

Q: long long자료형과 double 자료형의 차이점
S: long long 자료형은 정수형에서 8바이트까지 사용가능 (%lld) // 모든 정수값에 대해 정확한 표현 가능
   double자료형은 실수형에서 8바이트까지 사용가능  (%lf) // 실수 일부 값에서 정밀도 손실이 발생가능

Q:NZEC오류
S: scanf 함수에서 배열이 아닌 형식문자를 가지고 올 경우에 &사용하지 않으면 발생

Q: c언어 비교법
S: a == b == c와 같이 연속된 비교는 C 언어에서는 사용할 수 없습니다.
 대신, a == b && b == c와 같이 각각의 비교를 분리해야 합니다.
